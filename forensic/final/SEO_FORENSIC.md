# SEO FORENSIC — Spirit Revival Africa
**Audit Date:** 2025  
**Scope:** usePageMeta coverage, robots.txt, sitemap, canonical URLs, CSR limitations, content discoverability

---

## 1. Technology SEO Baseline

SRA is a **Client-Side Rendered (CSR) React application**. All HTML is generated by JavaScript at runtime. This has significant SEO implications:

- **Google:** Can index CSR content but with a delay (days to weeks). Crawl budget is spent on re-crawling JS-rendered pages.
- **Bing/DuckDuckGo:** Limited JS rendering support. May not see dynamic content at all.
- **Social media crawlers (Facebook, Twitter, WhatsApp):** Do NOT execute JavaScript. They read raw HTML only. This means all per-page OG tags set by `usePageMeta` are invisible to social media scrapers — only the static defaults in `index.html` are seen.
- **SEO recovery comment in code:** `usePageMeta.js` has a comment: `"For full SSR-level crawlability, migrate to Next.js or implement prerendering (see SEO_RECOVERY_PLAN.md)"` — `SEO_RECOVERY_PLAN.md` does not exist in the repo.

## 3. robots.txt Analysis

**File:** `frontend/public/robots.txt`

```
User-agent: *
Allow: /

Disallow: /api/
Disallow: /messages
Disallow: /dashboard
Disallow: /profile
Disallow: /verify-email
Disallow: /reset-password

Sitemap: https://spiritrevivalafrica.com/sitemap.xml
```

**Assessment:**
- ✅ `/api/` correctly blocked — saves crawl budget, prevents data exposure
- ✅ `/dashboard`, `/messages`, `/profile` blocked — auth-gated, no indexable content
- ✅ `/verify-email`, `/reset-password` blocked — utility pages
- ✅ `Sitemap:` directive present with correct absolute URL
- ⚠️ `/stories/submit` is crawlable (not blocked) — correct, it's a public page, but without content it wastes crawl budget
- ℹ️ `/admin/` is not explicitly blocked. Django admin is crawlable (though rate-limited). Should add `Disallow: /admin/` for good practice.

## 4. sitemap.xml Analysis

**File:** `frontend/public/sitemap.xml`  
**Type:** Static, manually maintained

```xml
11 URLs included:
/ (weekly, priority 1.0, lastmod 2026-03-29)
/book/beneath-the-crown (monthly, priority 0.9)
/content (daily, priority 0.8)
/prayer (daily, priority 0.8)
/groups (weekly, priority 0.7)
/hubs (weekly, priority 0.7)
/discipleship (weekly, priority 0.7)
/worship (monthly, priority 0.6)
/gallery (weekly, priority 0.6)
/register (monthly, priority 0.5)
```

**Issues:**
- ❌ No individual story URLs — every approved ShortStory has its own route `/stories/:id` but none are in the sitemap. If 100 stories are approved, none are individually crawlable via sitemap.
- ❌ Static file — not updated when new stories are approved or content is published. Requires manual maintenance.
- ❌ `lastmod` dates are hardcoded to `2026-03-29` — incorrect/misleading to search engines.
- ✅ Priority and changefreq values are reasonable and differentiated.
- ✅ Namespace declared correctly.
- ⚠️ `/login` is excluded (correct). `/profile/settings` excluded (correct).

**Recommendation:** Generate the sitemap dynamically from Django using a view at `/sitemap.xml` that queries approved ShortStory records. Use `django.contrib.sitemaps` or serve from frontend if URLs are predictable.

## 5. Canonical URL Analysis

### Static canonical in index.html
```html
<link rel="canonical" href="https://spiritrevivalafrica.com/" />
```
This hardcoded canonical points ALL pages to the homepage. When Google sees `/prayer`, `/content`, `/stories/123` — each has a canonical pointing to `/`. This tells Google: "all these pages are duplicates of the homepage."

**This is incorrect and actively harmful to SEO.** It suppresses ranking for all non-homepage URLs.

**`usePageMeta` does NOT update the canonical tag.** No code in the repo updates `<link rel="canonical">` per page.

## 9. Recommendations

### P0 — Fix the canonical tag
Update `usePageMeta.js` to also set canonical:
```js
let canonical = document.querySelector('link[rel="canonical"]');
if (!canonical) {
  canonical = document.createElement('link');
  canonical.rel = 'canonical';
  document.head.appendChild(canonical);
}
canonical.href = window.location.href; // or a canonical base URL
```

### P0 — Add usePageMeta to StoryPage
```jsx
usePageMeta({
  title: story.title,
  description: story.story?.slice(0, 155),
  // If story has a photo, pass it as ogImage
});
```

### P1 — Create og-image.png
1200×630px brand image. Referenced in all OG and Twitter card meta tags but missing from `/public/`.

### P2 — Add Disallow: /admin/ to robots.txt

### P2 — Dynamic sitemap
Use Django `sitemap` framework to generate `/sitemap.xml` dynamically including approved ShortStory URLs.

### P3 — Add JSON-LD to key pages
At minimum: `Organization` on homepage, `Article` on story pages, `Book` on book page.

### P4 — Prerendering
Consider `vite-plugin-prerender` or migration to Next.js for social media crawlability.
